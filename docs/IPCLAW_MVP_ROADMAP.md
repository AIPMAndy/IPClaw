# IPClaw MVP Roadmap (30 Days)

本路线图按“马斯克五步法”执行：先删需求，再做最小闭环，再加速迭代。

## North Star（首 90 天）

- GitHub Star `1000+`
- GitHub Followers `300+`
- 可联系线索 `200+`
- 每月案例复盘 `4`

## Week 1: Baseline Parity（先稳）

- Freeze CLI scope: only keep `ip:run` pipeline + templates + docs.
- Ensure `npm run lint/test/build` all pass on clean install.
- Publish `v0.1.0-ipclaw-alpha` only after P0 pass.
- 明确 V1 禁做项：不做全自动发布，不做复杂面板。

## Week 2: Positioning + Content（先有产物）

- Add positioning prompts and output schema.
- Add content repurposing templates (GitHub + short posts).
- 发布首个“输入→输出”完整示例。
- 在仓库内提供可复用模板入口。

## Week 3: Distribution + Approval Flow（先控风险）

- Add channel adaptation templates.
- Add mandatory human approval step before outbound publish.
- Add CTA and tracking parameters.
- 每条分发内容必须可回链到 GitHub。

## Week 4: KPI Loop（形成飞轮）

- Add weekly KPI snapshot generation.
- Add keep/drop/double-down recommendation summary.
- Publish one case-study style changelog.
- 固定每周复盘：保留 / 下线 / 加倍投入。
