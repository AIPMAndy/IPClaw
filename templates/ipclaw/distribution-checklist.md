# IPClaw Distribution Checklist（分发与审批卡）

## A. 发布前检查

- [ ] 这条内容是否明确服务一个目标人群？
- [ ] 是否包含可验证结果（截图/数据/案例）？
- [ ] 是否包含唯一 CTA（只保留一个主要动作）？
- [ ] 是否附带追踪参数（来源平台、活动名、日期）？
- [ ] 是否通过人工审批（语气、事实、风险）？

## B. 渠道适配

- [ ] GitHub：更新 README/Release/Discussion 至少其一。
- [ ] X / 公众号：提炼单一核心观点，附回链。
- [ ] 社区（Discord/论坛）：问题导向，引导讨论。

## C. 发布后 24 小时动作

- [ ] 记录曝光、点击、互动、转化。
- [ ] 回复高价值评论与问题。
- [ ] 归档可复用话术到模板库。

## D. 风险守则

- [ ] 禁止无审批自动外发。
- [ ] 禁止在内容中暴露密钥或隐私。
- [ ] 禁止夸大收益或保证式表达。
